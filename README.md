# 新能源汽车售后运营操作手册

> 面向主机厂售后运营、区域售后管理与门店服务团队的售后运营体系化手册。  
> 当前版本：**v2.3 表格修正版**

## 1. 项目定位

本仓库用于维护《新能源汽车售后运营操作手册》的 GitHub 可读版本。

这份手册不是单纯的售后服务流程说明，而是围绕新能源汽车售后运营建立一套可执行、可检查、可复盘的运营体系，覆盖：

- 主机厂售后运营中心如何制定规则、指标、SOP 和审计机制
- 区域售后团队如何穿透门店问题、推动整改和复盘
- 门店服务团队如何按统一口径执行接待、维修、配件、客户沟通和风险上报
- 新能源汽车特有的三电、OTA、远程诊断、预测性维护、客户承诺 SLA 等运营场景

## 4. 手册结构概览

手册主体分为 15 章：

1. **定义与边界**：说明售后运营是什么、管什么、不管什么
2. **组织与岗位职责**：明确总部、区域、门店及关键岗位分工
3. **业务层操作 SOP**：覆盖维修保养、三电、充电、救援、OTA、增值服务、客户服务、销售交付到售后交接
4. **运营层操作 SOP**：覆盖服务网络、工单、配件、技师、客户、财务、质量、数据、体验、风险合规
5. **指标体系总表**：建立业务层与运营层核心指标
6. **诊断流程与升级机制**：说明异常识别、分级、升级和闭环
7. **指标诊断卡**：围绕核心指标解释异常、原因、联动指标和解决动作
8. **专项 SOP 与合规细则**：覆盖安全事件、三包、召回、数据合规
9. **主机厂售后运营增强机制**：客户分级、KPI、技师认证、客户旅程、系统工具、岗位能力、知识库、审计、客户承诺 SLA
10. **软件与智能化售后 SOP**：覆盖 OTA、车机、智驾、远程诊断、软件争议和研发回流
11. **电池全生命周期管理 SOP**：覆盖电池拆装、储运、SOH、梯次利用、车电分离、延保和风险红线
12. **售后经营管理与预算 SOP**：覆盖预算、产值拆解、费用审批、结算、复盘和经营仪表盘
13. **售后变更与发布管理 SOP**：覆盖变更分级、灰度发布、回退、新车型上市和价格工时变更
14. **补强专题 SOP**：覆盖索赔、事故车、热失控、VOR、VOC、外包、数据治理、大客户/车队
15. **附录**：术语表、模板清单、检查清单、汇报口径、版本记录和指标速查表

## 6. 使用建议

### 快速阅读

如果只想快速理解框架，建议先看：

- 第 1 章：定义与边界
- 第 3 章：业务层操作 SOP
- 第 4 章：运营层操作 SOP
- 第 5 章：指标体系总表
- 第 6 章：诊断流程与升级机制
- 第 9.10 节：客户承诺与 SLA 管理机制

### 落地执行

如果用于真实运营落地，建议按以下顺序推进：

1. 先明确组织职责和 RACI
2. 再统一业务 SOP 与 SLA 口径
3. 建立核心指标看板
4. 用诊断卡处理异常指标
5. 建立升级、复盘和审计机制
6. 最后补齐软件、电池、经营、变更和专项场景

### 后续精简方向

当前版本偏完整，适合作为知识底稿和体系化手册。后续如要做减法，可进一步压缩为：

- 管理层速览版
- 区域督导执行版
- 门店服务经理版
- 两页速查版
- 指标诊断卡独立版

## 7. 维护规则

建议后续维护遵循以下原则：

- **先改 Markdown，再导出 Word / PDF**：保证 GitHub 上能看到完整修改历史
- **新增内容必须进目录**：避免正文有新增、目录看不到
- **该表格化的内容必须用表格**：尤其是 SLA、权限、指标、流程、RACI
- **少写解释，多写动作**：手册应指导执行，不堆概念
- **英文简称首次出现要注释**：例如 SLA（Service Level Agreement，服务级别承诺）
- **每次版本更新要写版本说明**：说明新增、删除、合并和重大调整
