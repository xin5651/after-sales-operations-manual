# 第 12 章 售后经营管理与预算 SOP


## 12.0 本章定位与适用边界

为什么单独成章：

主手册第 4.6 节"财务与商业运营"主要讲毛利率监控与异常排查，未覆盖"预算编制、产值拆解、费用审批、跨部门结算、经营复盘"等运营负责人级别的经营管理动作。

售后运营负责人最终对 P&L（损益表）负责，"会运营但不会算账"无法在主机厂或大型经销商集团中胜任。

本章建立"年度预算 → 月度看板 → 异常审批 → 跨部门结算 → 周/月/季复盘"的完整经营管理闭环。

适用边界：

覆盖：年度预算编制、月度产值/成本/毛利/费用四维看板、异常费用审批、跨部门结算、经营复盘节奏。

不覆盖：财务部门内部的总账、税务、审计、合并报表（属财务部职能）。

## 12.1 年度售后预算编制 SOP

定位：年度预算是售后运营全年经营的"指挥棒"，决定网络扩张、人员招聘、备件投入、营销资源的优先级。预算编制必须自下而上 + 自上而下双向收口。

| 编制阶段 | 关键动作 | Owner | 时间节点 |
| --- | --- | --- | --- |
| 1. 战略输入 | 总部下发新一年销量目标、保有量预测、网络规划、新车型上市计划 | 战略 / 销售 → 售后 | T-3 月 |
| 2. 自下而上预测 | 各区域 / 门店基于历史数据 + 增长假设上报收入 / 成本预测 | 门店 → 区域 → 总部 | T-2 月 |
| 3. 自上而下校准 | 总部按战略要求与历史规律校准上报数据，识别保守 / 激进项 | 售后运营中心 | T-2 月末 |
| 4. 预算评审 | 财务 + 业务联席评审；锁定收入目标、成本上限、费用预算、人力编制、CapEx | 财务 + 售后高管 | T-1 月 |
| 5. 预算下发 | 正式预算下发各区域 / 门店；签订年度经营责任书 | 售后运营中心 | T-1 月末 |
| 6. 预算锁定 | 预算锁定后不轻易调整；调整需走预算变更流程（详见 13.5） | 财务 + 售后高管 | 全年 |

预算编制核心结构（六大模块）：

| 预算模块 | 关键科目 | 主要驱动因子 |
| --- | --- | --- |
| 收入预算 | 维修工时收入 / 配件收入 / 三包索赔收入 / 增值服务收入 / 延保收入 | 保有量 × 进店率 × 客单价 |
| 成本预算 | 直接人工 / 配件成本 / 索赔成本 / 救援外包 / 客户补偿 | 收入结构 × 各项成本率 |
| 费用预算 | 门店运营费用 / 营销费用 / 培训费用 / 系统费用 / 差旅费用 | 门店数 × 单店费用 + 集中费用 |
| 人力预算 | 总部 / 区域 / 门店人员编制；招聘 / 离职 / 培训计划 | 工位数 / 车流量 / 服务标准 |
| CapEx | 门店建设 / 工位扩建 / 设备升级 / 系统建设 | 战略规划 + 网络扩张计划 |
| 专项储备 | 应急响应储备金 / 重大事件公关储备 / 法务争议储备 | 历史风险事件规模 × 概率 |

## 12.2 月度产值目标拆解 SOP（金字塔法）

定位：年度收入预算必须拆解到月、店、车型、工单类型，才能在月度异常时定位到具体维度。"只看总数"是经营失明。

产值拆解金字塔（自上而下五层）：

| 层级 | 维度 | 示例 / 用途 |
| --- | --- | --- |
| L1 | 年度总收入 | 全年售后总收入 100M |
| L2 | 按季度 / 月份 | 季节因子分配；旺季 Q1/Q4 占比高 |
| L3 | 按区域 / 城市级 | 一线城市单店产值 > 三线城市 |
| L4 | 按门店 | 每店年度收入责任书；与店总绩效挂钩 |
| L5 | 按工单类型 | 维修保养 / 三电 / 钣喷 / 增值各占比例 |

门店产值测算公式（建议在 12.1 预算编制中使用）：

| 因子 | 含义 | 影响逻辑 |
| --- | --- | --- |
| 有效工位数 | 可同时作业的工位数（含快保 / 综合 / 高压 / 钣喷） | 产值天花板 |
| 工位利用率 | 工位实际作业小时 ÷ 工位可用小时 | 反映经营效率 |
| 客单价 | 单台进店车辆平均售后产值 | 反映客户结构与服务深度 |
| 进店次数 | 保有量 × 年均进店频次 | 反映客户活跃度 |
| 公式 | 工位数 × 利用率 × 单工位日产能 × 营业天数 | 年度产值预测 |
| 校准 | 与历史同店数据回归校准，剔除异常店和新店爬坡 | 降低预测偏差 |

| 指标 | 建议口径 | 观察频率 | 异常信号 |
| --- | --- | --- | --- |
| 月度产值达成率 | 实际产值 ÷ 月度产值目标 | 月 | 低于 90% 红灯 |
| 门店产值排名 | 门店产值绝对值 + 同比 / 环比 + 达成率综合排名 | 月 | 连续 3 月尾部需约谈 |
| 工单类型结构变化 | 各工单类型收入占比环比变化 | 月 | 剧烈变化代表客户结构或服务策略变化 |

## 12.3 异常费用审批权限矩阵

定位：客户补偿、特批折扣、紧急采购、外包加急等费用必须有清晰的审批权限，避免"门店随意承诺、总部背锅"。

| 费用类型 | 门店店总 | 区域总监 | 售后总经理 | 财务总监 | 总经理 |
| --- | --- | --- | --- | --- | --- |
| 客户补偿（单笔） | ≤ 2000 | ≤ 1万 | ≤ 5万 | ≤ 10万 | > 10万 |
| 特批折扣（单笔） | ≤ 工时 20% | ≤ 工时 40% | ≤ 工时 60% | 联审 | 联审 |
| 紧急配件采购 | ≤ 5000 | ≤ 5万 | ≤ 20万 | ≤ 50万 | > 50万 |
| 外包加急费用 | ≤ 3000 | ≤ 2万 | ≤ 10万 | 联审 | 联审 |
| 公关 / 媒体补偿 | 不允许门店决策 | 联审 + 法务 | 联审 + 法务 | 联审 + 法务 | 联审 + 法务 |
| A 类客诉应急 | 不允许门店决策 | 现场授权 | 事后报备 | 事后报备 | 事后报备 |

审批红线：

🔴 严禁拆单：超过权限的费用拆成多笔避开审批。

🔴 严禁事后补单：超权限费用必须事前审批；除 A 类客诉应急外不允许"先支付后补流程"。

🔴 严禁口头承诺：所有费用承诺必须有书面 / 系统记录，避免门店人员变动后无据可查。

🟡 月度审计：财务 + 售后联合每月抽审异常费用单据，发现违规追责到人。

## 12.4 跨部门结算 SOP（销售 → 售后 / 市场 → 售后）

定位：销售部门承诺的"购车赠送 X 次保养"、"赠送电池包检测"、市场部门发起的"老客户专享活动"等，最终都在售后门店执行，必须有清晰的结算机制，否则售后变成"白干活"。

| 场景 | 结算方向 | 执行流程 | 注意事项 |
| --- | --- | --- | --- |
| 销售赠送保养 / 检测 | 销售部 → 售后部 | 销售在交付时录入系统；售后核销时按内部结算价计入 | 内部结算价需财务事前确认 |
| 销售赠送代步车 / 接送 | 销售部 → 售后部 | 按实际发生工时 + 物料成本结算 | 需事前约定服务边界 |
| 市场活动赠送服务 | 市场部 → 售后部 | 活动方案需售后预审；按方案约定结算 | 禁止市场单方面承诺超出售后能力的服务 |
| 延保 / 服务包 | 客户购买 → 售后部 | 售后按延保产品定价收取；递延确认收入 | 售后产品的设计需财务+精算+售后联评 |
| 保险公司送修 | 保险公司 → 售后部 | 按合作协议结算工时 / 配件 / 折扣率 | 保险费率需法务+财务签字 |
| 厂家服务活动 | 厂家 → 区域 → 门店 | 按厂家结算政策；门店上报实际工单 | 关键件回收 + 拍照留痕 |

所有跨部门承诺必须有书面 SLA 或合作协议，禁止口头约定。

内部结算价（销售→售后、市场→售后）每年至少 review 一次，避免"售后亏本服务"或"销售薅羊毛"。

月度结算未达成的，由财务发起对账会议，3 个工作日内闭环。

| 指标 | 建议口径 | 观察频率 | 异常信号 |
| --- | --- | --- | --- |
| 跨部门结算月结率 | 当月跨部门结算单当月完成 ÷ 应结算单 | 月 | 低于 95% 排查流程瓶颈 |
| 售后接单内部毛利率 | 内部结算收入 - 直接成本 / 内部结算收入 | 季 | 持续亏损需调整内部结算价 |
| 销售口头承诺投诉率 | 客户投诉销售承诺与实际不符 ÷ 售后总投诉 | 月 | 上升需协同销售培训 |

## 12.5 预算变更 / 调整 SOP

定位：年度预算锁定后不轻易调整。但当出现重大事件（新车型延迟上市、政策变化、重大召回、市场环境剧变等）时，必须有规范的预算变更流程，避免"年中追加 / 砍预算"导致执行混乱。

| 变更类型 | 触发条件 | 审批权限 | 生效要求 |
| --- | --- | --- | --- |
| 预算增加（< 5%） | 局部业务调整、临时项目 | 售后总经理 | 财务备案 + 季度评审 |
| 预算增加（5-15%） | 业务结构变化、新车型上市 | 售后总经理 + 财务总监 | 联签 + 总经理知会 |
| 预算增加（> 15%） | 重大召回、应急响应、战略调整 | 总经理联审 | CEO 签字 + 董事会知会 |
| 预算压减 | 收入显著低于预测、成本失控 | 财务总监发起 + 售后总经理执行 | 同步下发各区域 / 门店 |
| 预算结构调整 | 科目间调剂（不影响总额） | 售后总经理 + 财务备案 | 不需要 CEO 审批 |

## 12.6 经营复盘节奏 SOP（周/月/季/年）

定位：经营复盘不是"开会汇报"，而是从数据 → 结论 → 动作 → 跟踪闭环的完整管理动作。不同节奏复盘的颗粒度和目的不同。

| 频率 | 复盘范围 | 参与方 | 核心产出 |
| --- | --- | --- | --- |
| 周复盘 | 本周收入 / 成本 / 关键指标 / 异常事件 / 下周动作 | 区域级 + 门店级 | 周经营快报 + TOP 3 改善动作 |
| 月复盘 | 月度预算达成 / 红黄绿灯指标 / 重大事件 / 下月计划 | 总部 + 区域 + 门店 | 月度经营报告 + 责任书 |
| 季复盘 | 季度 P&L / 预算偏差分析 / 战略动作进度 / 人才盘点 | 总部 + 区域高管 | 季度经营复盘报告 |
| 年复盘 | 全年 P&L / 战略达成 / 组织 / 人才 / 新一年规划 | 全集团 / 全板块 | 年度经营报告 + 新一年战略输入 |

复盘四步法（每次复盘强制走完）：

① 看数据：用统一口径的看板拉数；不允许"自带数据"。

② 找原因：按 5 Why 拆解；不允许"因为客户少了"这种顶层归因。

③ 定动作：每个红灯指标必须有 Owner + 截止时间 + 验收标准。

④ 跟闭环：下周 / 下月复盘第一件事是看上次动作执行情况。

## 12.7 售后经营仪表盘标准（四维）

定位：经营仪表盘是经营层最常看的"驾驶舱"。必须用一张看板回答"现在做得好不好、问题在哪、要改什么"。

| 维度 | 核心指标 | 看板呈现要求 |
| --- | --- | --- |
| 收入维度 | 收入总额 / 同比 / 环比 / 预算达成率 / 客单价 / 进店次数 | 红黄绿灯 + 趋势线 |
| 成本维度 | 直接人工成本率 / 配件成本率 / 索赔成本率 / 外包成本率 | 与基准对比 + TOP 异常 |
| 毛利维度 | 总毛利率 / 按工单类型 / 按门店毛利率 / 按车型毛利率 | 热力图 + TOP 异常 |
| 费用维度 | 门店运营费用率 / 营销费用率 / 培训费用率 / 应急储备金使用率 | 与预算对比 + 异常预警 |

看板治理红线：

🔴 数据口径必须统一：所有看板使用同一指标字典，禁止"看自己版本数据"。

🔴 数据延迟必须标注：每张看板顶部显示数据更新时间 + 数据源。

🟡 看板变更必须留痕：口径变更需版本号 + 生效日期 + 知会业务方。

## 12.8 售后经营管理专项风险红线

🔴 预算红线：年度预算锁定后不允许超支执行；超预算必须事前走变更审批。

🔴 审批红线：禁止拆单避审批、事后补审批、口头承诺无书面留痕。

🔴 财务红线：客户补偿、特批折扣、紧急采购等所有异常费用必须在系统中留痕，禁止现金账外操作。

🔴 数据红线：经营看板使用统一指标口径，禁止区域 / 门店"自带数据"汇报。

🟡 跨部门红线：销售 / 市场承诺的服务必须有 SLA 协议，禁止口头约定。

🟡 复盘红线：复盘动作必须有 Owner + 截止时间 + 验收标准，禁止"会而不议、议而不决、决而不行"。

## 12.11 本章落地评审清单

以下清单用于下一轮版本更新、落地前评审和管理层复盘。它不是草稿问题，而是正式的“待确认/待深化”事项池。

1. 异常费用审批权限中的金额阈值是否需要按企业规模、区域层级和门店类型重新校准。

2. 月度产值测算公式是否需要补充按车型、店型、城市级别、工位类型的修正系数表。

3. 周 / 月 / 季 / 年经营复盘是否需要增加标准会议议程、数据包模板和行动项追踪表。

4. 跨部门结算机制是否适用于当前组织形态；单店模式、经销商集团、主机厂直营应分别简化或扩展。

5. S-33 ~ S-40 经营类新增指标是否并入第 5 章指标体系总表，或保留为经营管理子看板。
