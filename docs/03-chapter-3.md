# 第 3 章 业务层操作 SOP


本章按售后业务层 7 大模块，逐一给出标准操作流程（SOP）。每个模块统一包含：模块定义、适用场景、标准流程、关键节点与 SLA、异常处置、责任矩阵。一线门店以本章为日常执行依据，区域督导以本章为稽核标准。

## 3.1 维修保养

模块定义

覆盖常规保养、故障维修、机电维修、钣金喷漆、事故维修、召回与服务活动，确保车辆在全生命周期内保持安全可靠的运行状态。

适用场景

客户预约 / 到店保养（首保、定期保养、里程保养）。

故障报修（机械 / 电气 / 电控）。

事故车维修（含保险理赔协同）。

厂家发起的召回 / 服务活动批量到店。

远程诊断触发的主动保养建议到店。

标准流程

客户通过 APP / 400 / 门店预约，系统自动匹配工位与技师并确认时间。——服务顾问，≤ 2 小时确认。

客户到店，服务顾问完成接车检查（环车检查 + 电子健康报告），登记客户需求并生成工单。——服务顾问，≤ 15 分钟。

系统根据故障类型、技师技能矩阵、工位状态自动派工，复杂故障标记"需远程诊断支持"。——工单系统 / 服务经理，≤ 5 分钟。

技师按标准作业指导书（SOP Checklist）执行维修 / 保养，全程拍照留痕，关键工序扫码确认。——技师，按工时定额。

质检员按质检清单逐项验收，新能源车型增加高压系统复检项。——质检员，≤ 20 分钟。

服务顾问向客户解释维修内容和费用，客户确认后系统推送电子报告，交车并提醒下次保养周期。——服务顾问，≤ 10 分钟。

系统在交车后 24 小时内触发回访（短信 / APP / 电话），记录客户评价并关联工单。——客服 / 系统，≤ 24 小时。

关键节点与 SLA

| 关键节点 | SLA 标准 | 责任人 |
| --- | --- | --- |
| 预约确认 | ≤ 2 小时响应客户 | 服务顾问 |
| 接车建单 | 到店后 ≤ 15 分钟完成 | 服务顾问 |
| 派工 | 建单后 ≤ 5 分钟完成 | 系统 / 服务经理 |
| 维修作业 | 按工时定额，偏差 ≤ 10% | 技师 |
| 质检交付 | ≤ 20 分钟 | 质检员 |
| 客户交车 | 全流程 ≤ 承诺时间 | 服务顾问 |
| 回访闭环 | 交车后 ≤ 24 小时 | 客服 |
| 一次修复率（FFR） | ≥ 92% | 技师 / 质量运营 |

异常处置

触发条件：维修超时（超工时定额 30% 以上）

动作：技师报工单系统标记异常，服务顾问主动联系客户更新 ETA（预计完成时间），超 4 小时提供代步车或补偿方案。

升级路径：门店服务经理 → 区域运营经理（超 8 小时）→ 总部业务运营（超 24 小时）。

触发条件：配件缺货导致停工

动作：立即触发跨店 / 跨仓调拨，同步给客户 ETA；超 48 小时启动应急直发。

升级路径：配件员 → 区域仓 → 中央仓 / 供应链。

触发条件：质检不合格

动作：退回技师返工，记录返工原因，返工超 2 次升级技术支持。

升级路径：质检员 → 技师主管 → 区域技术支持。

触发条件：客户投诉维修质量

动作：24 小时内安排复检，指定高级技师处理；确认为己方问题的免费返修并补偿。

升级路径：服务经理 → 区域督导 → 总部质量运营（A 类客诉）。

| 关键活动 | R 负责 | A 最终负责 | C 协同 | I 知会 |
| --- | --- | --- | --- | --- |
| 预约排程 | 服务顾问 | 服务经理 | 技师主管 | 客服 |
| 接车检查与建单 | 服务顾问 | 服务经理 | 技师 | 客户运营 |
| 派工与维修作业 | 技师 | 服务经理 | 配件员 / 质检员 | 服务顾问 |
| 质检验收 | 质检员 | 技师主管 | 技师 | 服务经理 |
| 交车与回访 | 服务顾问 / 客服 | 服务经理 | 质量运营 | 区域运营 |
| 异常升级 | 服务经理 | 区域运营经理 | 总部业务运营 | 售后总监 |

## 3.2 三电专属服务

模块定义

专门面向电池、电机、电控（合称"三电"）以及高压系统和充电系统的检测、诊断、维修与更换服务，是新能源汽车售后的核心差异化模块。

适用场景

电池 SOH（State of Health，健康度）定期检测与评估。

电池故障维修（模组级 / 整包级更换）。

电机、电控系统故障诊断与维修。

高压系统绝缘检测、高压线束与连接器维护。

BMS（Battery Management System，电池管理系统）数据异常排查。

充电系统（车载充电机 OBC、DC-DC 变换器）故障处理。

标准流程

远程预诊断：车云平台拉取 BMS 数据、故障码、SOH 快照，三电技术支持初步判断故障方向。——三电技术支持，≤ 30 分钟。

高压安全确认：车辆到店后，持证高压技师按《高压安全作业规程》执行断电、放电、绝缘检测，确认安全后方可拆检。——高压技师，≤ 20 分钟。

三电深度检测：使用专用检测设备完成电池单体电压、内阻、温度一致性、绝缘电阻等项目检测，生成标准化检测报告。——三电技师，按检测项目定额。

维修 / 更换决策：三电技术支持（或总部专家台远程会诊）根据检测数据判定故障根因，明确维修策略（模组维修 / 整包更换 / 电控软件刷写）和保修责任归属。——三电技术支持 / 专家台，≤ 2 小时。

维修执行：按策略完成维修或更换，作业全程影像留痕，关键件（电池包、电机总成）做故障件回收标记。——三电技师，按工时定额。

复检与交付：完成高压系统复检（绝缘、连接器力矩、冷却管路密封），试车验证后交付客户并推送电子检测报告。——质检员 / 服务顾问，≤ 30 分钟。

数据回流：故障数据、检测报告、维修记录回写车辆档案，典型案例进入知识库，批量问题触发 QRQC（Quick Response Quality Control，快速响应质量控制）。——三电技术支持 / 数据运营，≤ 48 小时。

关键节点与 SLA

| 关键节点 | SLA 标准 | 责任人 |
| --- | --- | --- |
| 远程预诊断 | 接单后 ≤ 30 分钟出初步判断 | 三电技术支持 |
| 高压安全确认 | 到店 ≤ 20 分钟完成断电+绝缘 | 高压技师 |
| 三电检测报告 | 检测完成后 ≤ 1 小时出报告 | 三电技师 |
| 维修策略确认 | 检测报告后 ≤ 2 小时 | 三电技术支持 / 专家台 |
| 电池包到件时长 | ≤ 72 小时（常规）/ ≤ 24 小时（应急） | 配件运营 / 供应链 |
| 高压复检 | 维修完成后 ≤ 30 分钟 | 质检员 |
| 故障数据回流 | ≤ 48 小时进入车辆档案 | 数据运营 |
| 高压安全合规率 | ≥ 99% | 安全负责人 |

异常处置

触发条件：高压安全检查不合格（绝缘电阻 < 安全阈值）

动作：立即隔离车辆至专用高压工位，禁止非持证人员接触；启动高压应急预案，通知安全负责人。

升级路径：高压技师 → 门店服务经理 → 区域安全负责人 → 总部安全（涉及人身安全立即上报）。

触发条件：电池整包更换等件超 72 小时

动作：启动应急直发，同步为客户提供代步车或出行补贴，每日主动更新到件 ETA。

升级路径：配件员 → 区域配件运营 → 中央仓 / 供应链负责人。

触发条件：模组维修后 30 天内返修

动作：全量复查初次维修数据和检测报告，由专家台重新判定根因；若为同一故障升级为整包更换。

升级路径：三电技师 → 三电技术支持 → 总部专家台 / 质量运营。

触发条件：保修责任争议（客户 vs 厂家 vs 供应商）

动作：收集完整证据链（BMS 日志、充电记录、使用环境、检测报告），48 小时内出具责任判定书。

升级路径：三电技术支持 → 质量运营 → 法务（涉诉或金额 > 5 万元）。

责任矩阵（RACI）

| 关键活动 | R 负责 | A 最终负责 | C 协同 | I 知会 |
| --- | --- | --- | --- | --- |
| 远程预诊断 | 三电技术支持 | 门店服务经理 | 数据运营 | 区域运营 |
| 高压安全检查 | 高压技师 | 门店服务经理 | 安全负责人 | 三电技术支持 |
| 三电检测与报告 | 三电技师 | 三电技术支持 | 质量运营 | 服务顾问 |
| 维修策略与保修判定 | 三电技术支持 / 专家台 | 质量运营 | 法务 | 服务经理 |
| 电池包 / 关键件调拨 | 配件员 | 配件运营 | 供应链 | 服务顾问 |
| 故障数据回流与 QRQC | 三电技术支持 | 质量运营 | 研发质量 | 数据运营 |

## 3.3 充电相关服务

模块定义

覆盖家用充电桩安装、家充桩售后维修以及公共充电故障协助，打通"买车—装桩—用桩—报障"全链路的充电服务闭环。

适用场景

新车交付后家用充电桩安装（含物业协调、电力报装、现场施工）。

家充桩日常故障报修与定期巡检。

公共充电桩使用中的兼容性问题、充电中断、计费异常等客户求助。

因充电问题引发的客诉升级处理。

标准流程

安装需求确认：交车时或通过 APP 提交家充桩安装申请，客服确认安装地址、电力条件、物业许可。——客服 / 交付专员，≤ 24 小时。

现场勘察：安排服务商上门勘察安装条件（电表容量、走线距离、墙体打孔、接地条件），出具勘察报告并报价。——安装服务商，预约后 ≤ 3 个工作日。

物业与电力协调：协助客户与物业沟通安装许可，指导客户完成电力增容 / 独立电表报装。——充电服务专员 / 安装服务商，≤ 5 个工作日。

安装施工：按《家充桩安装标准》完成安装、接线、调试、通电测试，拍照留痕并提交验收报告。——安装服务商，≤ 1 个工作日（施工当天）。

客户验收：服务顾问与客户确认安装质量，指导充电操作，客户签署验收单。——服务顾问 / 安装服务商，≤ 30 分钟。

售后报修响应：客户通过 APP / 400 报修，系统自动派单至对应服务商，24 小时内完成远程诊断或上门维修。——充电服务专员 / 服务商，≤ 24 小时。

公共桩协助闭环：客户报告公共桩问题，客服记录桩编号和故障现象，联系桩运营商推进处理并 48 小时内回复客户。——客服 / 充电服务专员，≤ 48 小时。

关键节点与 SLA

| 关键节点 | SLA 标准 | 责任人 |
| --- | --- | --- |
| 安装申请确认 | ≤ 24 小时响应 | 客服 / 交付专员 |
| 现场勘察 | 预约后 ≤ 3 个工作日 | 安装服务商 |
| 安装完成（下单到通电） | ≤ 30 个自然日 | 充电服务专员 |
| 安装一次通过率 | ≥ 90% | 安装服务商 |
| 家充桩故障响应 | 报修后 ≤ 24 小时 | 充电服务专员 / 服务商 |
| 家充桩二次返修率 | ≤ 5% | 服务商 / 质量运营 |
| 公共桩报障闭环 | ≤ 48 小时回复客户 | 客服 / 充电服务专员 |

异常处置

触发条件：物业拒绝安装或电力条件不满足

动作：充电服务专员协助客户与物业 / 供电局二次沟通，提供标准化安装说明函和安全承诺书；仍无法解决的推荐共享桩或公共桩解决方案。

升级路径：充电服务专员 → 区域客户运营 → 总部充电业务（涉及批量物业拒装纳入专项治理）。

触发条件：安装后 7 天内故障

动作：服务商 24 小时内上门免费排查修复；若为施工质量问题，扣减服务商绩效并安排复检。

升级路径：服务商 → 充电服务专员 → 质量运营。

触发条件：公共桩兼容性批量问题（同车型多例）

动作：收集故障桩编号、故障码、车辆 VIN 和充电日志，联合桩运营商和车辆研发排查兼容性，48 小时内出具临时解决方案。

升级路径：充电服务专员 → 总部充电业务 → 研发 / 桩运营商。

触发条件：充电安全事件（过温 / 过充 / 漏电保护触发）

动作：立即指导客户断电并远离车辆，记录事件信息，2 小时内安排技术人员到场排查。

升级路径：客服 → 安全负责人 → 总部安全 / 法务（涉及人身或财产损失）。

| --- | --- | --- | --- | --- |
| 安装需求确认与勘察 | 客服 / 安装服务商 | 充电服务专员 | 交付专员 | 区域运营 |
| 物业 / 电力协调 | 充电服务专员 | 区域客户运营 | 安装服务商 | 客服 |
| 安装施工与验收 | 安装服务商 | 充电服务专员 | 质量运营 | 服务顾问 |
| 家充桩售后维修 | 服务商 | 充电服务专员 | 质量运营 | 客服 |
| 公共桩问题协助 | 客服 | 充电服务专员 | 桩运营商 | 区域运营 |
| 充电安全事件处置 | 安全负责人 | 总部安全 | 法务 | 充电服务专员 |

## 3.4 道路救援

模块定义

为行驶中或停放中出现故障、事故、电量耗尽等无法自行移动的车辆提供现场救援和转运服务，含新能源特有的电量焦虑救援（移动充电车 / 拖至最近充电桩）。

适用场景

车辆抛锚（三电故障、机械故障、轮胎爆胎等）。

电量耗尽（趴窝）需移动充电车或拖车至充电点。

交通事故现场救援与拖车。

涉水 / 高温 / 极寒等极端场景下的特殊救援。

高压系统故障导致的安全隔离与转运。

标准流程

客户报案：客户通过 APP 一键救援 / 400 电话报案，系统自动定位车辆 GPS 位置并记录故障描述。——客户 / 客服，实时。

派单匹配：系统根据车辆位置、故障类型、救援资源分布自动派单至最近的合作救援服务商。——系统 / 救援调度，≤ 5 分钟。

救援确认：服务商接单后确认预计到达时间（ETA），系统自动推送 ETA 给客户并开启实时位置共享。——服务商 / 系统，≤ 3 分钟。

现场救援：救援人员到达现场，评估故障和安全状况（新能源车必须先确认高压系统状态），执行现场修复或拖车。——救援人员，视场景而定。

安全转运：需拖车的新能源车辆必须使用平板拖车（禁止牵引），高压故障车辆按《高压车辆转运规程》执行隔离转运。——救援人员，≤ 2 小时完成转运。

到店交接：车辆到达门店后，救援人员与服务顾问完成交接，车辆信息、故障描述、现场照片录入工单。——救援人员 / 服务顾问，≤ 15 分钟。

回访与数据回流：救援完成后 4 小时内回访客户满意度，救援数据（场景 / 原因 / 时长 / 成本）回写车辆档案。——客服 / 数据运营，≤ 4 小时。

关键节点与 SLA

| 关键节点 | SLA 标准 | 责任人 |
| --- | --- | --- |
| 报案响应 | ≤ 1 分钟接线 + ≤ 5 分钟派单 | 客服 / 救援调度 |
| 救援到达（城区） | ≤ 45 分钟 | 救援服务商 |
| 救援到达（郊区 / 高速） | ≤ 90 分钟 | 救援服务商 |
| 救援完成（现场修复或启动拖车） | 到达后 ≤ 30 分钟 | 救援人员 |
| 拒单 / 改派 | ≤ 3 分钟完成改派 | 救援调度 |
| 救援满意度 | ≥ 85% | 客服 / 质量运营 |
| 二次救援率 | ≤ 3% | 质量运营 |
| 救援数据回流 | ≤ 4 小时写入档案 | 数据运营 |

异常处置

触发条件：服务商拒单或超时未接单

动作：系统 3 分钟内自动改派；连续两次改派失败升级人工调度，同步通知客户最新 ETA。

升级路径：系统 → 救援调度 → 区域运营经理。

触发条件：高压系统故障车辆现场救援

动作：救援人员禁止接触高压部件，仅做安全隔离（警戒带 + 绝缘手套确认无外露高压），联系三电技术支持远程指导后使用平板拖车转运。

升级路径：救援人员 → 三电技术支持 → 安全负责人。

触发条件：客户等待超过承诺 ETA

动作：每超时 15 分钟自动推送延误通知，超 30 分钟客服主动致电安抚并更新 ETA，超 60 分钟启动补偿预案。

升级路径：系统 → 客服 → 区域运营经理。

触发条件：二次救援（首次救援未解决）

动作：原救援工单标记"二次救援"，指定高级服务商或门店技师到场，事后做根因分析并纳入服务商考核。

升级路径：救援调度 → 质量运营 → 区域运营经理。

## 3.5 OTA 升级服务

模块定义

管理车辆 OTA（Over-The-Air，空中下载）远程软件升级的全流程，包括升级推送、执行监控、失败回滚、升级后问题响应，确保升级安全落地且不影响客户用车体验。

适用场景

厂家发布新版本 OTA 升级包（功能升级 / BUG 修复 / 安全补丁）。

客户自助触发 OTA 升级。

OTA 升级失败需要远程或到店恢复。

升级后客户反馈功能异常或体验下降。

监管或安全要求的强制升级。

标准流程

升级前准备：研发完成升级包测试并发布，产品运营确认升级范围（车型 / 批次 / 区域）、风险等级、回滚方案。——研发 / 产品运营，发布前 ≥ 3 个工作日。

用户告知：通过 APP 推送 / 短信 / 车机弹窗向目标用户发送升级通知，说明升级内容、预计时长、注意事项（建议 WiFi + 电量 > 50%）。——产品运营 / 客服，升级推送前 ≥ 24 小时。

升级推送与执行：系统按批次灰度推送，车辆在满足条件（驻车 + 电量 + 网络）后自动下载并安装，升级过程中车云平台实时监控状态。——系统 / 数据运营，按灰度计划。

升级结果确认：系统自动采集升级结果（成功 / 失败 / 超时），失败车辆自动标记并触发回滚或人工介入。——系统 / OTA 运营，实时。

失败回滚：升级失败的车辆自动执行回滚至上一稳定版本；回滚失败的立即联系客户安排到店恢复。——系统 / 客服 / OTA 运营，回滚 ≤ 30 分钟。

升级后监控：升级完成后 7 天内持续监控升级车辆的故障码、客诉、进店率，发现异常立即暂停后续批次推送。——数据运营 / OTA 运营，持续 7 天。

问题响应与闭环：客户反馈升级后问题，客服 2 小时内响应，判断是否为升级引发，是则推送修复方案或安排到店处理。——客服 / OTA 运营，≤ 2 小时。

关键节点与 SLA

| 关键节点 | SLA 标准 | 责任人 |
| --- | --- | --- |
| 用户告知触达率 | ≥ 95% | 产品运营 / 客服 |
| OTA 升级成功率 | ≥ 95% | OTA 运营 / 研发 |
| 升级失败回滚成功率 | ≥ 97% | 系统 / OTA 运营 |
| 回滚失败到店恢复 | 联系客户 ≤ 2 小时 | 客服 / OTA 运营 |
| 升级后 7 天客诉率 | ≤ 0.5% | OTA 运营 / 质量运营 |
| 关键 BUG 修复发版 | ≤ 7 个工作日 | 研发 |
| OTA 售后联动响应 | ≤ 2 小时 | 客服 / OTA 运营 |

异常处置

触发条件：灰度阶段升级失败率 > 1%

动作：立即暂停后续批次推送，研发排查根因；已推送未安装的车辆撤回推送通知。

升级路径：OTA 运营 → 研发 → 产品运营 → 售后总监（影响面 > 1000 台）。

触发条件：回滚失败（车辆无法正常使用）

动作：客服 30 分钟内联系客户，安排拖车到店恢复；到店后通过有线刷写恢复至稳定版本。

升级路径：客服 → OTA 运营 → 救援调度 → 研发。

触发条件：升级后批量客诉（同一问题 ≥ 10 例）

动作：暂停推送 + 启动专项调查，48 小时内出具根因报告和修复方案；已升级车辆推送热修复或安排到店。

升级路径：OTA 运营 → 质量运营 → 研发 → 售后总监。

触发条件：安全类升级（涉及制动 / 转向 / 高压）延迟推送

动作：升级为 P0 优先级，总部每日跟踪覆盖率直至 100%，未升级车辆逐一电话通知。

升级路径：OTA 运营 → 产品运营 → 安全负责人 → 总部管理层。

## 3.6 增值服务

模块定义

在基础维保之外，为客户提供代步车、代驾接送、美容贴膜改装、保险代办、理赔协助、二手车检测置换等衍生服务，提升客户粘性和单客产值。

适用场景

维修超时（≥ 4 小时）或配件等待期间需要代步车 / 出行补贴。

客户选择上门取送车服务（代驾接送）。

客户到店做美容、贴膜、轻改装等项目。

新车保险代办或续保推荐。

事故车理赔协助与定损对接。

客户有置换需求，需二手车检测评估。

标准流程

需求识别：服务顾问在接车环节或客户主动咨询时识别增值需求，系统根据客户画像主动推荐。——服务顾问 / 系统，接车时。

方案推荐与报价：根据客户需求匹配服务项目，提供标准报价和套餐方案，客户确认后生成增值服务工单。——服务顾问，≤ 10 分钟。

服务执行： - 代步车：确认可用车辆，办理借用手续，客户签署使用协议。 - 美容贴膜：技师按标准施工，完工后质检拍照。 - 保险代办：录入客户信息，对接保险公司出单。 - 二手车评估：检测师完成车况检测，出具评估报告。 ——对应责任人，按项目时限。

交付与确认：服务完成后通知客户验收，推送服务报告和费用明细，客户确认后结算。——服务顾问，≤ 30 分钟。

回访与复购引导：服务完成后 48 小时内回访满意度，推送会员积分或优惠券引导复购。——客服 / 客户运营，≤ 48 小时。

## 3.7 客户服务（前后台）

模块定义

覆盖 400 客服中心、在线客服、售后预约、投诉处理、回访闭环等前后台客户服务触点，确保客户在售后全旅程中的每个接触点都能获得一致、高效、有温度的响应。

适用场景

客户首次进线咨询（维保预约、费用查询、服务网点查询等）。

客户投诉受理与处理（服务质量、维修质量、费用争议、等待过长等）。

售后预约（保养、维修、充电桩安装等）。

维修 / 服务完成后的客户回访。

多渠道（电话 / APP / 微信 / 车机）客户信息统一管理。

舆情 / 社交媒体负面反馈的主动拦截与处理。

标准流程

客户进线 / 在线接入：400 / APP / 微信 / 车机渠道统一接入，系统自动识别客户身份（VIN + 手机号），弹屏显示车辆信息和历史服务记录。——客服 / 系统，≤ 20 秒接入。

需求分类与处理：客服根据客户需求分类（咨询 / 预约 / 投诉 / 救援），简单问题当场解答，复杂问题创建工单转派。——客服，≤ 5 分钟。

投诉分级与派单：投诉按严重程度分为 A（重大）/ B（一般）/ C（轻微）三级，A 级立即升级门店服务经理和区域运营，B/C 级 4 小时内派单处理。——客服 / 质量运营，≤ 30 分钟。

投诉处理与跟踪：责任门店在规定时限内联系客户、确认事实、给出解决方案，处理过程全程记录在系统。——门店服务经理 / 服务顾问，A 级 ≤ 4 小时 / B 级 ≤ 24 小时 / C 级 ≤ 48 小时。

客户确认与关单：解决方案执行完毕后，客服回访确认客户满意，客户确认后关闭工单。应避免未经客户确认的"假关单"。——客服，≤ 24 小时。

回访执行：维修 / 服务完成后 24 小时内触发回访（系统自动 + 人工补充），记录 NPS（Net Promoter Score，净推荐值）评分和问题反馈。——客服 / 系统，≤ 24 小时。

回访问题闭环：回访中发现的问题创建新工单跟踪，48 小时内给出处理方案并回复客户。——客服 / 门店，≤ 48 小时。

## 3.8 销售交付到售后交接 SOP 【v2.3新增】

定位：很多售后投诉来自销售交付阶段的口径不清或权益漏录。主机厂应把交付承诺结构化写入系统，售后按同一口径承接。

| 1. 权益确认 | 交付前确认保养、延保、服务包、充电桩、软件订阅、BaaS（Battery as a Service，电池即服务）等权益。 | 销售交付/售后政策 | 权益清单录入 CRM/DMS |
