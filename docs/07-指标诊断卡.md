# 第 7 章 指标诊断卡


## 7.1 诊断卡使用说明

指标诊断卡是售后运营日常管理的核心工具。每张卡对应第 5 章中的一个核心指标，当该指标出现异常时，使用者可以快速定位问题、找到原因、明确解决路径和责任人。

什么时候用：当某个指标连续偏离目标值（通常 2-3 个统计周期），或单次大幅波动时，立即翻到对应诊断卡。

怎么用：按卡上 6 个字段依次走——先确认异常含义，再看可能原因，然后检查关联指标是否同步异常，最后按解决步骤分配任务。

谁用：区域售后运营负责人、门店服务经理、质量运营、客户运营等一线管理角色。总部职能部门用于专项分析和跨区复盘。

本章共 29 张诊断卡，分为业务层（16 张）和运营层（13 张），与第 5 章指标体系一一对应。每张卡包含以下 6 个标准字段：

是什么：一句话定义，含计算方式

异常是什么意思：升高/下降代表什么

为什么会异常：列出常见原因

会带哪些指标一起异常：列出关联指标编号和名称

怎么解决：按真实操作顺序的解决步骤

谁负责：列岗位/部门，含升级路径

## 7.2 业务层诊断卡（16 张）

### B-01 一次修复率 FFR

是什么：一台车一次进店把所有问题修好，且 30 天内不因相同问题再来的工单占比。计算方式：（无返修工单数 ÷ 同期总维修工单数）× 100%。

异常是什么意思：FFR 下降 = 没诊断对、没修好、或配件/质检出了问题。客户体验和返修成本同时恶化。

为什么会异常：

客户描述和故障现象没有结构化记录，技师凭经验猜修

远程诊断数据、故障码、BMS 日志没有进入工单

派工技能与问题类型不匹配，复杂三电问题派给非认证技师

配件没有一次配齐，导致二次返工

只修表面不修根因，质检走过场

OTA 已修复的软件问题门店仍按硬件方案处理

会带哪些指标一起异常：B-02 30 天返修率、O-03 配件首次到位率、O-05 技师认证覆盖率、O-10 QRQC 问题闭环率、O-08 NPS 净推荐值

怎么解决：

1. 按门店、车型、技师、故障族群切片，找异常集中点，不看平均数自嗨

2. 抽取 FFR 失败工单，检查四项：客户描述完整度、故障复现记录、远程诊断结论、影像留痕

3. 缺哪项补哪项：系统设必填校验、SOP 补判定树、远程诊断强制场景

4. 按问题类型分发 Owner：诊断错 → 售后技术支持；配件错 → 配件运营；作业错 → 技师运营加带教；软件类 → 接 OTA 版本简报

5. 对受影响客户指定唯一 Owner，给统一口径和补偿方案

6. 每周 QRQC 复盘 TOP 故障族群，关闭标准 = 下周期 FFR 改善

谁负责：门店服务经理 → 区域售后运营 → 质量运营；升级到区域售后总监。

### B-02 30 天返修率

是什么：车辆交付后 30 天内因相同或关联问题再次进店的工单比例。计算方式：（30 天内同故障返修工单数 ÷ 同期交付工单数）× 100%。

异常是什么意思：返修率升高 = 首次维修没有真正解决问题，或根因没有被识别。客户多跑一趟，成本和不满同步上升。

为什么会异常：

首修诊断时客户描述、故障码、复现条件记录不完整

远程诊断结论没有进入工单，技师没参考车云数据

只换件不排根因，尤其三电类偶发故障

质检标准不明确或走过场，没发现遗留问题

故障件未回收检测，无法追溯真实原因

售后发现的问题没有进入 QRQC 闭环，同类问题反复出现

会带哪些指标一起异常：B-01 一次修复率 FFR、O-10 QRQC 问题闭环率、O-08 NPS 净推荐值、B-16 投诉 48h 闭环率

怎么解决：

1. 按门店、车型、故障族群、技师切片，定位返修集中点

2. 抽返修工单对比首修诊断结论和二次故障证据，找断点

3. 缺诊断记录的补 SOP 和系统必填校验；缺影像的补留痕要求

4. 把问题归类：诊断错、作业错、配件错、软件错，分派给对应 Owner

5. 对受影响客户给统一解释、进度同步和补偿口径

6. 下周期复查返修率是否改善，未改善升级到区域负责人

谁负责：质量运营 → 门店服务经理 → 售后技术支持；升级到区域售后运营。

### B-03 单车年进店次数

是什么：每台在册车辆每年到店接受服务的平均次数。计算方式：年度工单总数 ÷ 在册车辆总数。

异常是什么意思：偏低 = 客户流失到外部修理厂或认为新能源不需要保养；偏高 = 可能存在返修多、过度维保推销或质量问题。

为什么会异常：

保养提醒没有触达或触达太晚，客户错过保养窗口

客户认为新能源车不需要定期保养，认知偏差未纠正

预约到店转化率低，提醒发了但客户没来

外部修理厂截流，价格或便利性更优

返修频繁导致数据虚高，不是真正的服务需求增长

会带哪些指标一起异常：B-14 增值服务渗透率、O-07 客户流失预警响应率、O-08 NPS 净推荐值、O-09 单店月均毛利率

怎么解决：

1. 按车龄、里程、城市、门店切片，找出低进店和高进店的人群

2. 对低进店客户分层触达：保养提醒 + 权益告知 + 预约入口一次给齐

3. 针对新能源车主设计保养认知教育内容（底盘、空调、刹车等仍需保养）

4. 对高进店客户排查是否因返修导致，如是则归因到 B-01/B-02 处理

5. 每月看逾期客户回流率，不只看消息发送量

谁负责：客户运营 → 门店服务顾问 → 区域售后运营。

### B-04 维修工时达成率

是什么：实际维修工时与标准工时的比值，反映维修效率。计算方式：（实际完成标准工时总量 ÷ 可用工时总量）× 100%。

异常是什么意思：偏低 = 产能闲置、等件空转或派工不合理；偏高 = 赶工降质量或工时定额不合理。

为什么会异常：

预约到店波峰波谷不匹配，技师忙闲不均

工单等件或等审批导致技师空转

派工不按技能匹配，高技能技师做低价值活

工时定额未考虑新能源车型特殊工序（高压断电、绝缘检测等）

绩效只看数量不看质量，技师赶工

工位或设备不足限制产出

会带哪些指标一起异常：B-01 一次修复率 FFR、O-02 工单 SLA 达成率、O-05 技师认证覆盖率、O-09 单店月均毛利率

怎么解决：

1. 按门店、技师等级、工单类型拆工时达成率，不只看平均数

2. 低达成先查四类原因：无活、等件、等审批、工位不足

3. 高达成同时看返工率和客诉率，防止以量换质

4. 复杂三电工单留给认证技师，基础保养做快保分流

5. 排班按预约预测和历史峰谷动态调整

6. 绩效加入质量权重：返工率、FFR、客户评价

谁负责：门店服务经理 → 技师运营 → 区域售后运营。

### B-05 三电故障 48h 闭环率

是什么：三电类故障工单在 48 小时内完成诊断、维修并交付的比例。计算方式：（48h 内闭环的三电工单数 ÷ 三电工单总数）× 100%。

异常是什么意思：下降 = 三电诊断能力不足、等件太久、或高压作业资质/工位受限，客户车辆长时间不可用。

为什么会异常：

SOH 评估口径不统一，诊断环节反复确认

BMS 数据、故障码没有进入工单，技师盲修

高压认证技师不足，复杂工单排队

电池包等大件交付周期长

整包更换与模组维修判定边界不清，决策时间长

三电专家台响应慢，门店等技术支持

会带哪些指标一起异常：B-06 SOH 检测覆盖率、B-07 高压安全合规率、O-04 大件平均交付时长、O-06 高压作业资质合规率、O-10 QRQC 问题闭环率

怎么解决：

1. 按车型、电池供应商、故障码、门店切片定位瓶颈

2. 把 BMS 数据、故障码、充电行为、温度曲线纳入必填诊断包

3. 明确整包换、模组修、软件修的判定树，减少决策犹豫

4. 电池包等大件建单即锁库，48h 内给客户 ETA 和替代出行

5. 三电专家台设限时响应 SLA，超时自动升级

6. TOP 批次问题进入 QRQC 并追踪到供应商改善

谁负责：三电技术支持 → 门店高压技师 → 质量运营；升级到质量负责人和供应商质量。

### B-06 电池健康度 SOH 检测覆盖率

是什么：需要做电池健康评估的车辆中，实际完成 SOH 检测并留存结果的比例。计算方式：（完成 SOH 检测车辆数 ÷ 应检测车辆数）× 100%。

异常是什么意思：下降 = 三电健康判断缺少基础证据，保修争议和误判风险上升，电池安全隐患无法早期发现。

为什么会异常：

SOH 检测标准和口径不统一，门店执行不一致

检测设备或软件版本不兼容新车型

技师不知道哪些场景通常需要做 SOH（进店保养、事故后、保修前）

检测耗时长，快修工位不愿做

检测结果没有纳入系统，手工记录易遗漏

客户拒绝检测，门店未做说服

会带哪些指标一起异常：B-05 三电故障 48h 闭环率、B-07 高压安全合规率、O-10 QRQC 问题闭环率、O-11 远程诊断使用率

怎么解决：

1. 定义强制 SOH 检测场景：进店保养、电池类工单、事故维修、保修索赔、二手车评估

2. 工单建单时系统自动判断是否触发 SOH 检测，未做不宜交付

3. 统一检测设备和软件版本，确保各车型兼容

4. 检测结果自动回写车辆档案和工单，不依赖手工录入

5. 对低覆盖门店排查设备、培训和执行问题

谁负责：三电技术支持 → 门店高压技师 → 系统产品 → 质量运营。

### B-07 高压安全合规率

是什么：涉及高压系统的维修作业中，按标准完成断电、绝缘检测、防护穿戴、作业记录等合规步骤的比例。

异常是什么意思：下降 = 存在人身安全和车辆安全隐患，一旦发生事故后果极其严重，同时面临监管和法律风险。

为什么会异常：

高压作业 SOP 培训不足或流于形式

非认证技师违规接触高压系统

防护设备不齐全或过期未检

赶工省略绝缘检测和断电确认步骤

作业记录和影像留痕不完整，无法追溯

门店管理层对高压安全重视不够，未纳入日常检查

会带哪些指标一起异常：O-06 高压作业资质合规率、O-05 技师认证覆盖率、B-06 SOH 检测覆盖率、O-13 三包合规审计通过率

怎么解决：

1. 高压工单系统强制校验技师资质，无证不可派工

2. 作业前自动生成安全检查清单：断电确认、绝缘检测、防护穿戴、工具检查

3. 每个关键节点拍照留痕，未上传不宜进入下一步

4. 防护设备定期盘点，过期 / 缺失自动提醒采购

5. 对违规作业零容忍：首次停工整改 + 带教，二次取消资质

谁负责：门店安全负责人 → 技师运营 → 风险合规；升级到区域售后总监。

### B-08 充电桩安装一次通过率

是什么：家充桩安装在首次上门即完成安装并通电的比例。计算方式：（一次通过安装数 ÷ 上门安装总数）× 100%。

异常是什么意思：下降 = 客户提车后长时间无桩可用，安装返工增加成本，客户满意度急剧下降。

为什么会异常：

安装前勘查不充分，到现场才发现电力条件不满足

物业沟通失败，缺乏标准话术和证明材料模板

安装团队外包质量不稳定

桩体型号与车型或电表不兼容

安装配件备货不齐，现场缺材料

客户对安装条件预期不清，临时变更需求

会带哪些指标一起异常：B-09 充电桩故障 48h 修复率、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值

怎么解决：

1. 安装前强制完成远程预勘查：电力容量、走线距离、物业政策、桩位照片

2. 给安装团队配标准物业沟通话术和证明材料包

3. 常见失败场景做成判定树：电力不足怎么办、物业不同意怎么办

4. 安装配件按订单自动配货，不做现场采购

5. 安装服务商按一次通过率月度考核，连续不达标淘汰

谁负责：充电服务运营 → 安装服务商管理 → 区域售后运营。

### B-09 充电桩故障 48h 修复率

是什么：家充桩故障报修后 48 小时内完成修复的比例。计算方式：（48h 内修复工单数 ÷ 充电桩报修工单总数）× 100%。

异常是什么意思：下降 = 客户充电不便，可能转向公共桩或对品牌充电服务失去信心。

为什么会异常：

桩体故障与车端故障互相甩锅，责任界定慢

维修备件不齐，需要等配件

外包维修团队响应慢或技术能力不足

故障诊断需要远程数据但系统不支持

偏远地区服务覆盖不足

会带哪些指标一起异常：B-08 充电桩安装一次通过率、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值

怎么解决：

1. 明确桩体故障 vs 车端故障的快速判定流程，不让客户被踢皮球

2. 常见故障备件前置到区域仓，维修工单建单即锁库

3. 外包维修团队设响应时效 SLA，超时自动改派

4. 充电桩接入远程监控，故障自动报警并推送维修工单

5. 偏远地区建立移动维修覆盖或备用方案

谁负责：充电服务运营 → 维修服务商 → 区域售后运营。

### B-10 救援 45 分钟到达率

是什么：从客户报案到救援资源到达现场在 45 分钟内完成的比例。计算方式：（45 分钟内到达数 ÷ 救援总派单数）× 100%。

异常是什么意思：下降 = 客户在路边长时间等待，焦虑和不安全感剧增，尤其新能源趴窝场景（电量耗尽、高压故障）更敏感。

为什么会异常：

偏远区域救援网点覆盖不足

外包救援商接单慢或拒单

事故、电量耗尽、高压故障等场景没有分流调度规则

客服定位和车辆状态采集不完整，调度信息不准

恶劣天气或节假日救援容量不足

救援拖车与服务中心衔接慢，客户被踢皮球

会带哪些指标一起异常：B-11 二次救援率、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值、O-01 网络覆盖达标率

怎么解决：

1. 按城市、时段、场景、救援商切片找超时热点

2. 超时客户先给进度和 ETA，不让客户失联

3. 高频超时区域补签救援商或部署移动服务车

4. 事故 / 电量耗尽 / 高压故障三类场景分别固化调度路径

5. 救援结束后数据回流车辆健康档案和服务中心工单

6. 月度淘汰 SLA 连续不达标救援商

谁负责：道路救援运营 → 客服调度 → 区域售后运营 → 服务网络运营。

### B-11 二次救援率

是什么：首次救援后短期内（通常 7 天）因相同或关联原因再次需要救援的比例。

异常是什么意思：升高 = 首次救援没有真正解决问题，客户被反复折腾，信任严重受损。

为什么会异常：

首次救援只做临时处理（搭电、拖车），没有排查根因

救援人员不具备新能源车基本诊断能力

救援数据没有回流到服务中心，到店后重新排查

电量耗尽场景只充电没排查充电系统和 BMS 异常

救援后没有安排到店检查的跟踪流程

会带哪些指标一起异常：B-10 救援 45 分钟到达率、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值、B-01 一次修复率 FFR

怎么解决：

1. 按救援场景（电量 / 故障 / 事故）和首次处理方式切片分析

2. 高压或三电相关救援必须在现场做基础检查清单

3. 救援结束强制生成到店检查预约，系统自动跟踪是否到店

4. 救援数据（故障码、现场照片、处理方式）实时回流门店工单

5. 对二次救援客户给专人跟踪 + 到店优先接待

谁负责：道路救援运营 → 门店服务经理 → 区域售后运营。

### B-12 OTA 升级成功率

是什么：发起 OTA 升级后成功完成且车辆功能正常的比例。计算方式：（升级成功车辆数 ÷ 发起升级车辆总数）× 100%。

异常是什么意思：下降 = 软件发布质量、车辆条件筛选、用户告知或售后兜底存在断点，客户车辆功能受损。

为什么会异常：

版本灰度策略不准，高风险车型被纳入推送

车辆电量、网络、停放条件不满足升级要求

升级前用户告知不充分，客户中断升级操作

升级包与车机硬件版本不兼容

失败回滚 SOP 不清晰，门店不知如何补救

售后端不知道版本变更内容，无法准确诊断升级后问题

会带哪些指标一起异常：B-13 OTA 回滚成功率、B-15 客服首次响应时长 FRT、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值

怎么解决：

1. 按版本、车型、车机硬件、地区和失败码切片定位失败集中点

2. 高风险版本暂停扩大推送，先处理已失败车辆

3. 失败车辆自动生成售后待办并给门店处理指引

4. 给用户补发清晰告知：升级条件、预估时长、失败后处理方式

5. 研发输出版本变更清单和已知问题给售后和客服

6. 复盘失败 TOP 原因，调整灰度规则和回滚流程

谁负责：软件运营 → OTA 产品 → 售后技术支持 → 客服；升级到研发质量负责人。

### B-13 OTA 回滚成功率

是什么：OTA 升级失败后执行回滚操作并成功恢复到可用版本的比例。

异常是什么意思：下降 = 升级失败的车辆无法回退到可用状态，客户车辆长期功能异常或不可用，通常需要到店处理。

为什么会异常：

回滚 SOP 不清晰，门店和客服不知道操作步骤

部分车型或版本不支持自动回滚

回滚包未提前预置到车端

回滚后新旧版本数据不兼容，出现二次异常

研发和售后信息不同步，回滚指令下发延迟

会带哪些指标一起异常：B-12 OTA 升级成功率、B-15 客服首次响应时长 FRT、B-16 投诉 48h 闭环率、O-08 NPS 净推荐值

怎么解决：

1. 所有 OTA 版本通常需要预置回滚包和自动回滚触发条件

2. 明确回滚 SOP：远程触发 → 车端自动 → 到店人工，三级递进

3. 回滚后自动验证关键功能并生成状态报告

4. 不支持自动回滚的车型，系统自动生成到店工单并通知客户

5. 每个版本发布前回滚测试通常需要通过，否则不上线

谁负责：OTA 产品 → 软件质量 → 售后技术支持；升级到研发负责人。

### B-14 增值服务渗透率

是什么：到店客户中购买至少一项增值服务（延保、美容、保险代办、充电套餐等）的比例。

异常是什么意思：下降 = 增值收入减少，客户终身价值未被充分挖掘；但过高也可能意味着过度推销。

为什么会异常：

增值服务产品设计不贴合新能源车主需求

服务顾问不了解产品或缺少推荐话术

推销时机不当（客户投诉时推销、维修等待焦虑时推销）

定价与竞品对比无优势

增值服务交付体验差，口碑差影响后续转化

会带哪些指标一起异常：B-03 单车年进店次数、O-09 单店月均毛利率、O-08 NPS 净推荐值、O-07 客户流失预警响应率

怎么解决：

1. 按客户画像（车龄、里程、消费记录）设计分层推荐策略

2. 训练服务顾问在正确场景推荐：交付时、保养满意后、续保前

3. 应避免在客户投诉或等待焦虑时推销增值服务

4. 渗透率和客诉率同时看，过高渗透伴随客诉上升时需要刹车

5. 月度复盘增值服务客户评价，持续优化产品和流程

谁负责：客户运营 → 门店服务顾问 → 区域售后运营 → 商业运营。

### B-15 客服首次响应时长 FRT

是什么：客户通过任意渠道（电话、在线、APP）发起咨询或投诉后，客服首次有效响应的平均时长。

异常是什么意思：升高 = 客户等待时间变长，觉得没人管，情绪升级风险上升，尤其紧急场景（趴窝、安全问题）影响极大。

为什么会异常：

客服人力配置与来电峰谷不匹配

机器人误导客户，增加无效交互

多渠道没有统一排队，重复进线占资源

复杂问题缺二线支持，一线空转

知识库过期，客服查不到答案

投诉和普通咨询分流不准，紧急问题进了普通队列

会带哪些指标一起异常：B-16 投诉 48h 闭环率、O-08 NPS 净推荐值、B-10 救援 45 分钟到达率

怎么解决：

1. 按渠道、时段、问题类型切片找高 FRT 场景

2. 紧急场景（救援、安全、停驶）设快速通道，不排队

3. 客服排班按来电预测调整，增加高峰时段人力

4. 机器人无法解决时快速转人工，不让客户反复描述

5. 知识库每周更新，新版本/新政策发布后 24h 内同步

6. 复杂问题设二线限时响应，一线不空转

谁负责：客服运营 → 售后技术支持 → 系统产品 → 区域售后运营。

### B-16 投诉 48h 闭环率

是什么：投诉工单在 48 小时内给出方案、执行完成并经客户确认关闭的比例。

异常是什么意思：下降 = 投诉只是流转或关单，没有真正解决客户问题。长尾投诉积累后会升级到监管、媒体或法务。

为什么会异常：

首响慢，客户觉得没人管，情绪从不满升级到愤怒

责任部门互相退回或口径不一致

补偿权限不明确，一线不敢做决定

门店证据（工单、影像、检测结论）不全，无法定责

高风险投诉没有提前识别和升级机制

系统只记录关单，不验证客户是否真正满意

会带哪些指标一起异常：B-15 客服首次响应时长 FRT、O-08 NPS 净推荐值、O-13 三包合规审计通过率、O-10 QRQC 问题闭环率

怎么解决：

1. 所有投诉指定唯一 Owner、下一步动作和截止时间

2. A 类客诉（涉安全、法律、媒体）进红色队列，2 小时内给方案方向

3. 给一线明确补偿权限和升级边界，不让客户等审批

4. 补齐工单、影像、检测结论和沟通记录，用证据说话

5. 关闭标准改成客户确认 + 根因归类完成，不只看系统关单

6. 每周复盘升级投诉的第一个失控节点

谁负责：投诉运营 → 客户关系中心 → 门店服务经理 → 区域售后运营；A 类升级到法务/公关。

## 7.3 运营层诊断卡（13 张）

### O-01 网络覆盖达标率

是什么：在目标市场中，用户在一定半径（如 30km）内能触达至少一个服务网点的覆盖比例。

异常是什么意思：下降 = 部分区域客户无法便捷到店，救援响应慢，品牌体验在这些区域空白。

为什么会异常：

新交付城市服务网点开设滞后于交付节奏

授权服务中心退出或降级，留下服务空白

偏远区域投入产出不平衡，新建网点难

移动服务车利用率低，未能有效补充固定网点

直营与授权服务标准不一致，部分网点被投诉后暂停

会带哪些指标一起异常：B-10 救援 45 分钟到达率、O-02 工单 SLA 达成率、O-08 NPS 净推荐值、B-03 单车年进店次数

怎么解决：

1. 按城市保有量和服务半径画出覆盖热力图，找出空白区域

2. 优先在空白区签约授权服务中心或部署移动服务车

3. 新车交付前 90 天确保交付城市有达标服务网点

4. 对退出/降级网点启动应急替代方案，不让客户断服务

5. 月度复盘覆盖率与救援响应、客户投诉的关联

谁负责：服务网络运营 → 区域售后运营 → 售后战略规划。

### O-02 工单 SLA 达成率

是什么：接待、诊断、审批、等件、施工、质检、交付等各节点按时完成的工单比例。

异常是什么意思：下降 = 工单流程卡在某个节点，不一定是技师慢，可能是诊断排队、审批拖延、配件等待或交付结算卡住。

为什么会异常：

工单关键字段漏填或乱填，影响后续流转

节点 SLA 没有被系统监控，超时无人升级

各节点之间断点多，上游完成但下游未接收

紧急插队工单打乱正常排产

影像和证据留痕不足，质检退回导致返工

系统只记录状态，不推动下一步动作

会带哪些指标一起异常：B-01 一次修复率 FFR、B-04 维修工时达成率、O-03 配件首次到位率、O-08 NPS 净推荐值

怎么解决：

1. 把总流程拆成接待→诊断→审批→等件→作业→质检→交付七段，找最慢节点

2. 超时工单每天自动推给 Owner，通常需要填写下一步动作

3. 审批慢就设预审材料清单和超时自动升级

4. 等件慢就联动配件锁库和 ETA 实时同步

5. 高风险字段设必填和格式校验，源头减少返工

6. 每周复盘 TOP 超时节点和异常门店

谁负责：工单运营 → 门店服务经理 → 配件运营 → 区域售后运营。

### O-03 配件首次到位率

是什么：工单首次申领配件能一次配齐且版本正确的比例。计算方式：（一次配齐工单数 ÷ 有配件需求的工单总数）× 100%。

异常是什么意思：下降 = 客户被迫等件或二次进店，维修交付时间延长，FFR 和客户满意度同步恶化。

为什么会异常：

关键件预测没有结合保有量、故障率和季节分布

门店仓 / 区域仓 / 中央仓库存数据不同步，虚假可得

没有锁库机制，普通工单抢走紧急工单的件

大件（电池包、电驱）交付周期长且物流复杂

供应商交付不稳定

新旧车型件号替代关系没维护，发错件

会带哪些指标一起异常：O-04 大件平均交付时长、B-01 一次修复率 FFR、O-02 工单 SLA 达成率、B-16 投诉 48h 闭环率

怎么解决：

1. 按车型、SKU、门店、区域切片，定位是全网缺还是某门店调拨慢

2. 受影响在途工单按客户优先级排序：A 类客诉 > 停驶 > 常规

3. 高风险客户优先跨仓调拨 + 紧急直发 + ETA 实时同步客户

4. 关键件和大件工单建单即锁库，防止被其他工单抢走

5. 现场盘点异常 SKU，账实不符立即冻结避免错派

6. 缺件 TOP N 进入售后-供应链周复盘

谁负责：配件运营 → 区域仓/中央仓 → 门店配件员；升级到售后总监和供应链总监。

### O-04 大件平均交付时长

是什么：电池包、电驱等大件从确认需求到到店可施工的平均时间。

异常是什么意思：升高 = 大件供应链拖住客户车辆，停驶时间延长，代步成本和投诉同时上升。新能源大件比燃油车更敏感。

为什么会异常：

电池包预测没有结合保有量、故障率和地区分布

中央仓与区域仓调拨慢

电池等危险品运输有特殊要求，物流周期长

换件审批链条过长，需多级确认

电池包型号版本复杂，匹配易错

锁库规则缺失，资源被低优先级工单占用

会带哪些指标一起异常：O-03 配件首次到位率、B-05 三电故障 48h 闭环率、B-16 投诉 48h 闭环率、O-09 单店月均毛利率

怎么解决：

1. 按车型、电池型号、仓库、区域拆分等件时长

2. 先处理停驶、营运车、A 类客诉高风险客户

3. 确认在途件 ETA 并每日同步客户和门店

4. 建立电池包专属安全库存和优先级锁库

5. 审批、运输、入库、施工四段分别设 SLA

6. 长期缺件型号进入供应链月度专项

谁负责：配件运营 → 供应链物流 → 区域售后运营 → 门店服务经理。

### O-05 技师认证覆盖率

是什么：具备对应车型、等级或高压/三电资质的技师占全部需持证技师的比例。

异常是什么意思：下降 = 新车型和高风险维修能力跟不上，复杂工单无人可派，安全与质量风险上升。

为什么会异常：

新车型培训晚于车辆交付，首批车到店时技师还没认证

授权门店培训执行弱，线上考试过但实操不足

认证过期未复训，证书断层

人员流失导致认证技师减少

门店用未认证技师补位接高压工单

会带哪些指标一起异常：B-01 一次修复率 FFR、B-07 高压安全合规率、O-06 高压作业资质合规率、B-05 三电故障 48h 闭环率

怎么解决：

1. 按车型、城市、门店、等级列出证书缺口矩阵

2. 新车型上市前 60 天锁定首批交付城市认证名单并完成培训

3. 高压/三电工单设置系统级无证不可派工规则

4. 证书到期前 30 天自动提醒复训

5. 培训结果看实操通过率和后续返工率，不只看考试分数

6. 缺证门店限制承接高风险工单并补区域支援

谁负责：技师运营 → 培训认证 → 门店服务经理 → 区域售后运营。

### O-06 高压作业资质合规率

是什么：涉及高压系统的工单中，由持有效高压资质技师在合规条件下完成作业的比例。

异常是什么意思：下降 = 非持证人员操作高压系统或未按 SOP 执行，人身安全和法律风险极高。

为什么会异常：

高压资质认证滞后，持证技师不够用

系统未强制校验资质，人工绕过派工

防护设备不齐全或过期

门店为赶工期让非持证技师上手

复训和年审机制不完善

会带哪些指标一起异常：B-07 高压安全合规率、O-05 技师认证覆盖率、O-13 三包合规审计通过率

怎么解决：

1. 系统层面强制：高压工单自动校验技师资质，不合格不可派工

2. 每月盘点高压资质有效期，到期前 30 天推送复训

3. 防护设备纳入门店日检清单，缺失或过期当天补齐

4. 任何绕过资质的派工行为自动告警并上报区域

5. 季度高压安全专项检查，不达标门店限制承接

谁负责：技师运营 → 风险合规 → 门店安全负责人 → 区域售后运营。

### O-07 客户流失预警响应率

是什么：系统识别出的流失风险客户中，在预警有效期内完成至少一次有效触达和干预的比例。

异常是什么意思：下降 = 流失客户没有被拦截，客户默默离开，等发现时已经流失到外部修理厂或竞品。

为什么会异常：

流失预警模型不准，误报多导致一线不信任

预警触发后没有分配到具体的人和动作

干预话术和权益设计不吸引人

门店对流失客户不重视，只关注新增

唤回动作粗放，群发短信没有个性化

会带哪些指标一起异常：B-03 单车年进店次数、O-08 NPS 净推荐值、O-09 单店月均毛利率、B-14 增值服务渗透率

怎么解决：

1. 优化流失预警模型：结合进店频次、保养逾期、投诉未闭环、NPS 低分等多维度

2. 预警触发后系统自动分配 Owner 和截止时间

3. 分层干预：高价值客户电话关怀、中价值客户精准推送权益、一般客户短信提醒

4. 每月看唤回成功率和回流客户的后续消费，不只看触达量

5. 连续流失门店纳入区域重点督导

谁负责：客户运营 → 门店服务顾问 → 区域售后运营。

### O-08 NPS 净推荐值

是什么：愿意推荐品牌的客户占比（9-10 分）减去不愿推荐客户占比（0-6 分）。

异常是什么意思：下降 = 客户整体信任和口碑在变差，影响复购、转介绍和品牌声誉。NPS 是结果指标，根因通常埋在其他指标里。

为什么会异常：

一次没修好，客户反复跑

维修等待时间长，进度不透明

投诉没人闭环，客户被踢皮球

费用解释不清，觉得被宰

服务顾问态度或沟通差

三电或 OTA 问题反复出现

充电桩安装或售后服务拖延

会带哪些指标一起异常：B-01 一次修复率 FFR、B-16 投诉 48h 闭环率、O-07 客户流失预警响应率、O-09 单店月均毛利率、B-02 30 天返修率

怎么解决：

1. 先按客户旅程触点拆 NPS：预约、接待、维修、交付、回访、投诉

2. 把低分样本回连到具体工单和门店，不看平均数自嗨

3. TOP 低分原因分派到责任模块：质量、配件、客服、体验、价格

4. 低分客户 48 小时内二次触达，给解释、方案或补偿

5. 门店改善看低分原因下降，不只看回访分数上升

6. 月度复盘 NPS 与经营指标的联动：流失率、复购率、产值

谁负责：客户运营 → 体验运营 → 区域售后运营 → 门店服务经理。

### O-09 单店月均毛利率

是什么：单店售后月度收入扣除直接成本后的毛利占收入比例。计算方式：（月售后收入 - 直接成本）÷ 月售后收入 × 100%。

异常是什么意思：下降 = 收入结构、成本控制、定价策略或保修结算出了问题。长期下降威胁门店可持续经营。

为什么会异常：

保修/三包工单占比过高，挤压自费收入

工时定价偏低或折扣失控

配件成本上升但售价未调整

低毛利工单占用高价值产能和工位

代步车、救援、补偿等服务成本失控

授权门店结算规则不清晰

会带哪些指标一起异常：B-04 维修工时达成率、B-14 增值服务渗透率、O-03 配件首次到位率、O-07 客户流失预警响应率

怎么解决：

1. 按收入类型拆毛利：自费、保修、三包、保险、增值，找下降来源

2. 定位问题是收入掉、成本升、折扣多、补偿多还是结构变

3. 对低毛利高占用工单调整流程或定价

4. 保修成本异常门店抽查索赔真实性和维修策略

5. 统一折扣权限，超限通常需要审批

6. 月度复盘毛利率和客户体验，不建议只为利润牺牲满意度

谁负责：财务运营 → 门店总经理 → 售后运营 → 区域财务。

### O-10 QRQC 问题闭环率

是什么：售后发现的质量问题完成归因、措施制定、执行验证和正式关闭的比例。

异常是什么意思：下降 = 售后问题没有回到研发和供应商，故障会反复出现，保修成本持续上升。

为什么会异常：

售后问题没有按故障族群归因，零散处理

故障件未回收检测，缺少物证

质量问题没有回流研发或供应商，信息断层

QRQC 会议只开会不产出永久措施

改善措施没有验证效果就关闭

批次风险识别滞后，扩散后才反应

会带哪些指标一起异常：B-01 一次修复率 FFR、B-02 30 天返修率、O-08 NPS 净推荐值、O-13 三包合规审计通过率、B-05 三电故障 48h 闭环率

怎么解决：

1. 按车型、年款、供应商、故障码、地区切片找问题集中点

2. 故障件强制回收检测并形成检测结论

3. 售后、研发、供应商共用同一问题清单和跟踪系统

4. 每个 TOP 问题指定根因、措施、Owner 和截止时间

5. 关闭标准通常需要包含下周期指标改善，不是会议纪要签字

6. 连续未改善问题升级质量负责人

谁负责：质量运营 → 研发质量 → 供应商质量 → 售后技术支持。

### O-11 远程诊断使用率

是什么：售后工单中实际调用车云数据、故障码或远程诊断结论的比例。

异常是什么意思：下降 = 车有数据但售后没用上，诊断仍靠经验和直觉，新能源车的数据优势没有转化为诊断效率。

为什么会异常：

门店不知道什么场景该用远程诊断

系统入口复杂或权限不足

车云数据延迟、缺失或质量差

远程诊断结论不准，技师不信任、不采纳

工单字段没有强制关联诊断结果

模型没有根据一线反馈持续校准

会带哪些指标一起异常：B-01 一次修复率 FFR、B-05 三电故障 48h 闭环率、O-02 工单 SLA 达成率、O-12 维修进度推送覆盖率

怎么解决：

1. 把三电、软件、偶发故障列为强制远程诊断场景

2. 工单建单时自动拉取最近故障码和车辆状态，不依赖技师手动查

3. 技师可反馈诊断结论是否准确，用于模型校准

4. 低使用门店排查系统入口、权限、培训和管理问题

5. 数据质量问题回到车云链路修复，不让一线背锅

6. 使用率和 FFR 联动考核，避免为用而用

谁负责：数据运营 → 售后技术支持 → 系统产品 → 门店技师主管。

### O-12 维修进度推送覆盖率

是什么：维修过程中向客户推送进度、费用变更或关键节点信息的工单比例。

异常是什么意思：下降 = 客户不知道车修到哪了，焦虑催促客服，信任感下降，投诉防御能力减弱。

为什么会异常：

系统没有在关键节点自动触发推送

服务顾问手动推送，忙时忘记

推送内容模板不清晰，客户看不懂

费用变更没有实时通知客户确认

工单状态更新不及时，推送依据不准

会带哪些指标一起异常：O-08 NPS 净推荐值、B-16 投诉 48h 闭环率、O-02 工单 SLA 达成率

怎么解决：

1. 系统在接待、诊断完成、报价确认、开工、质检、交付六个节点自动推送

2. 费用变更实时通知客户并获取确认

3. 推送内容标准化：当前进度、预计完成时间、下一步是什么

4. 对等待超预期客户自动推送道歉 + 补偿规则

5. 每周看推送覆盖率与 NPS/客诉的关联

谁负责：体验运营 → 系统产品 → 门店服务经理 → 服务顾问。

### O-13 三包合规审计通过率

是什么：三包相关工单在内部或外部审计中，按法规和企业标准正确受理、判责、处理和留痕的通过比例。

异常是什么意思：下降 = 门店三包执行不一致，面临监管投诉、法务风险和品牌风险。新能源三包（尤其动力电池）更复杂。

为什么会异常：

一线不熟悉三包条款，尤其新能源特有条款（电池质保、电驱保修等）

保修、三包、自费边界混淆，判责不准

证据采集不完整（检测报告、影像、沟通记录、客户签收）

地区监管尺度不同，总部规则无法覆盖

客户解释话术不统一，引发预期冲突

系统没有强制校验关键材料

会带哪些指标一起异常：B-16 投诉 48h 闭环率、B-07 高压安全合规率、O-06 高压作业资质合规率、O-08 NPS 净推荐值

怎么解决：

1. 列出高风险三包场景：退换修、动力电池质保、重大安全故障、反复维修

2. 系统强制采集证据：检测报告、影像、沟通记录、客户签收

3. 门店使用统一判责模板和客户解释话术

4. 争议工单进入区域法务/三包专家预审

5. 对三包投诉高发门店做专项培训和抽检

6. 每月复盘监管投诉案例并更新规则库

谁负责：风险合规 → 法务 → 三包专员 → 门店服务经理 → 区域售后运营。
